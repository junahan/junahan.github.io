本文介绍如何使用 Emacs Org-mode + ox-hugo 撰写博客，并通过 Hugo + Netlify/Github Page 发布博客网站。

1 安装和配置

以下安装命令均为 Mac OS 命令，其他平台请参照相关工具。

1.1 安装 Hugo

1

brew install hugo

1.2 安装 Netlify 命令行工具

在 Mac OS 上执行 brew install netlify 命令可安装 netlify-cli 工具。或者通过 npm 安装：

1

npm install netlify-cli -g

1.3 认证和使用 Netlify

Netlify 使用访问令牌进行认证，要获得一个访问令牌，使用如下命令：

1

netlify login

跟着提示步骤，在 Netlify 上创建站点。认证完成后获得访问令牌，Netlify CLI 将该令牌存储在您的 {Home}/.netlify/config.json 文件里面。随后的 netlify 命令将自动使用该令牌。

2 创建本地 Hugo 站点

2.1 选择和配置 Hugo Theme

我选择使用一款 Even 主题。

2.2 安装主题

1
2

# 执行 git clone 命令将主题拉取进入 hugo 站点的 themes 目录
git clone https://github.com/olOwOlo/hugo-theme-even themes/even

2.3 升级主题

1
2

cd ./themes/even/
git pull

2.4 配置主题

在该主题的 exampleSite 目录下有一个 config.toml 文件，将这个 config.toml 文件复制到你的站点目录下，根据自己的需求更改即可。注意: 对于这个主题，你应该使用 post 而不是 posts，即 hugo new post/some-content.md。

3 使用 ox-hugo

3.1 配置 ox-hugo

仅需要如下配置即可：

1
2
3
4

;; ox-hugo config
(use-package ox-hugo
  :ensure t          ;Auto-install the package from Melpa (optional)
  :after ox))

3.2 发布模式

ox-hugo 支持两种发布模式：

  • One post per Org subtree (官方推荐模式) - 每个 Org 子节作为一个单独的 Markdown 文档发布。
  • One post per Org file - 每个 Org 文件作为一个单独的 Markdown 文档发布。

我习惯使用 One post per Org file 模式。

3.3 Hugo Headers

Header 是否必须 说明
HUGO_BASE_DIR 必须指定 用于指定发布 Markdown 文件的目录
HUGO_SECTION 可选 默认为 posts 目录，可以在文件中指定，如果没有指定，则使用配置 org-hugo-section 配置

3.4 发布快捷键

我使用 C-c C-e H h (发布 Org file 为 Markdown post) 来发布我的文档。也可以使用 C-c C-e H o (发布并打开 Markdown post)。

4 发布

4.1 发布到 Netlify

  1. 编辑 ORG 文档 - 注意将文档元数据头属性 #+hugo_draft: true 修改为 #+hugo_draft: false 以发布文档。
  2. 发布 MD 文档 - 在 Emacs ORG 模型下运行 C-c C-e H h 将 .org 发布为 .md 文档。
  3. 使用 Hugo 构建 - 在命令行中运行 hugo 命令构建静态网站并使用 hugo server 运行本地服务器以验证网站内容。
  4. 使用 netlify 发布 - 在命令行中运行 netlify deploy --prod 发布网站。

4.2 发布至 Github 站点

Github Page 功能允许将 {githubID}.github.io 为域名创建站点。结合使用 Hugo 发布静态网站至 Github 站点的步骤如下。

  • 登陆 Github，创建名字为 {githubID}.github.io 的公共仓库，相关详情，请参阅创建 Github Pages 站点。
  • 进入 Hugo 项目仓库目录（我的本地工作目录是 junahan-site），如果该目录已经有名字为 public 的目录，运行 rm -rf public 删除它。
  • 运行命令 git submodule add -b master git@github.com:<USERNAME>/<USERNAME>.github.io.git public 以创建 git 子模块。
  • 创建 deploy.sh 脚本，并运行 chmod +x deploy.sh 命令将其设置为可以执行。

#!/bin/sh

# If a command fails then the deploy stops
set -e

printf "\033[0;32mDeploying updates to GitHub...\033[0m\n"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public

# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site $(date)"
if [ -n "$*" ]; then
  msg="$*"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master
  • 运行 hugo 测试并在本地测试站点。
  • 运行 ./deploy.sh 发布站点至 Github。
